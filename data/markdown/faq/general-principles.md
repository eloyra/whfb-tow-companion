# General Principles - Frequently Asked Questions

### [If a specific natural roll triggers an effect, is that effect still triggered if the dice is re-rolled and the re-roll gives a different result?](#if-a-specific-natural-roll-triggers-an-effect-is-that-effect-still-triggered)

No. When you re-roll a dice, the previous roll is discarded. This includes any effects that roll may have triggered.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [If I mark the position of a unit then proceed to move it before putting it back where it was and moving it again, does this count as a take-back?](#if-i-mark-the-position-of-a-unit-then-proceed-to-move-it-before-putting-it)

Yes, absolutely.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [What happens if a Hit is rolled on the Scatter dice for an object that moves in a random direction, do I use the small arrow on the Hit icon to determine the direction?](#what-happens-if-a-hit-is-rolled-on-the-scatter-dice-for-an-object-that-moves)

Essentially, objects that move in a random direction scatter, as described on page 95. Such objects only move in the direction shown by the small arrow on the Hit icon if the rules tell you they do so. Otherwise, the object does not move.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [What happens if a scatter roll causes any part of an object to cross beyond the edge of the battlefield?](#what-happens-if-a-scatter-roll-causes-any-part-of-an-object-to-cross-beyond)

The object is removed from play.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [If a model is required to make a Leadership test using its own Leadership characteristic, can it use the Leadership of a different model, the General for example, instead?](#if-a-model-is-required-to-make-a-leadership-test-using-its-own-leadership)

No. When the rules state a model must make a Leadership test using its own Leadership, it must use its own Leadership.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [Some single use special rules and magic items require a Leadership test to use. Do they count as having been used even if this test is failed?](#some-single-use-special-rules-and-magic-items-require-a-leadership-test-to)

Yes.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [Is there a maximum beyond which characteristics cannot be modified?](#is-there-a-maximum-beyond-which-characteristics-cannot-be-modified)

Unless a rule specifies a maximum, no.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [Can a unit arrayed in Close Order or Open Order formation be one model wide?](#can-a-unit-arrayed-in-close-order-or-open-order-formation-be-one-model-wide)

Yes.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [If some of my opponent’s attacks cause multiple Wounds to be lost per unsaved wound they cause, in what order are those unsaved wounds applied to my unit?](#if-some-of-my-opponents-attacks-cause-multiple-wounds-to-be-lost-per-unsaved)

Apply unsaved wounds to a unit one at a time and one model at a time. When a model is reduced to zero Wounds on its profile, it is removed from play as a casualty (as described on page 102). If a model loses more Wounds than it has on its profile, the excess wounds are lost – they do not spill over onto other models in the same unit.

With this in mind, apply unsaved wounds that cause models to lose all of their remaining Wounds first (Killing Blows and Monster Slaying Blows, for example), then unsaved wounds that cause multiple Wounds to be lost (Multiple Wounds (X), for example), then, finally, unsaved wounds that cause a single Wound to be lost.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [If a unit with the Press of Battle rule has two fighting ranks, can I remove casualties from the second rank?](#if-a-unit-with-the-press-of-battle-rule-has-two-fighting-ranks-can-i-remove)

No. Models in the first fighting rank, the rank in base contact with the enemy, fall first as casualties.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [If a unit contains models of two different troop types, how do I determine its maximum rank bonus?](#if-a-unit-contains-models-of-two-different-troop-types-how-do-i-determine)

In such cases, the troop type of the majority of the models determines the unit’s maximum rank bonus.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [How does a Close Order unit of just one model act, is it still a Close Order formation, or does it act like a Skirmisher?](#how-does-a-close-order-unit-of-just-one-model-act-is-it-still-a-close-order)

A Close Order unit always acts as such, even if it only contains a single model.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [Does a unit count as being obscured when some of the models within it are behind others?](#does-a-unit-count-as-being-obscured-when-some-of-the-models-within-it-are)

No. A unit cannot be obscured from the enemy by itself.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2

### [Can different saves be combined together to improve their save value? For example, if a model has two Ward saves, can they be combined to give a single Ward save with a better Warding value?](#can-different-saves-be-combined-together-to-improve-their-save-value-for)

No, values given as a target number cannot be combined to lower the target number. If a model has two or more Ward saves, as per the example, simply use the best.

Source: Official Warhammer: The Old World FAQ & Errata - Version 1.5.2
